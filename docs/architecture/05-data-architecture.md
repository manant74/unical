## BDI Data Model (`current_bdi.json`)

The core data structure is the BDI framework JSON, persisted per session:

```json
{
  "domain_summary": "One-sentence domain description",
  "beneficiario": {
    "beneficiario_name": "User category label",
    "beneficiario_description": "2-sentence role and goal summary",
    "beneficiario_inference_notes": ["inference evidence 1", "..."]
  },
  "desires": [
    {
      "desire_id": "D1",
      "desire_statement": "What the beneficiary wants to achieve",
      "priority": "high|medium|low",
      "success_metrics": ["Measurable outcome 1", "..."],
      "context": "Situational background",
      "motivation": "Why this desire matters"
    }
  ],
  "beliefs": [
    {
      "subject": "Entity or concept",
      "definition": "What it is, why it matters, how it works",
      "semantic_relations": "typed_relation",
      "source": "Exact text citation for verifiability",
      "importance": 0.9,
      "confidence": 0.9,
      "related_desires": [
        {
          "desire_id": "D1",
          "relevance_level": "CRITICO|ALTO|MEDIO|BASSO",
          "definition": "How this belief supports the desire"
        }
      ]
    }
  ],
  "intentions": []
}
```

**Backward Compatibility**: Old format uses `"personas": [...]` instead of `"beneficiario"`. Pages handle both formats transparently.

## Relevance Levels (Desire-Belief Mapping)

| Level | Meaning |
|-------|---------|
| **CRITICO** | Without this belief, the desire cannot be satisfied |
| **ALTO** | Strongly supports the desire; important contextual knowledge |
| **MEDIO** | Useful supporting knowledge; indirectly related |
| **BASSO** | Peripheral; tangentially related |

## Session Directory Structure

```
data/sessions/{uuid}/
├── metadata.json          # name, tags, created_at, last_accessed, status
├── config.json            # llm_provider, llm_model, llm_settings, context
├── current_bdi.json       # live BDI data
└── chat_history/          # per-agent conversation logs
```

## Context Directory Structure

```
data/contexts/{normalized_name}/
├── context_metadata.json  # name, document_count, belief_count
├── belief_base.json       # pre-extracted beliefs from KB (generated by Knol)
└── chroma_db/             # persistent ChromaDB vector store
```

## ChromaDB Schema

- **Collection**: one per context, named by normalized context name
- **Distance metric**: cosine similarity
- **Embedding dimension**: 384 (paraphrase-multilingual-MiniLM-L12-v2)
- **Metadata per chunk**: `source` (filename/URL), `chunk_index`

## Caching Strategy

| Data | Cache Mechanism | Invalidation |
|------|----------------|-------------|
| System prompts | Python `@lru_cache` | `clear_cache()` or use_cache=False |
| Streamlit UI state | `st.session_state` | Page reload / session reset |
| ChromaDB clients | `st.session_state` | Manual `release_connections()` |
| Session metadata | Disk read on demand | Updated on write |
