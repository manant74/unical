## 3.1 LLMManager (`utils/llm_manager.py`)

**Purpose**: Single entry point for all LLM calls. Abstracts multi-provider logic.

**Responsibilities**:
- Initialize and manage Gemini and OpenAI API clients
- Route calls to the correct provider
- Inject RAG context and system prompts
- Expose model metadata for dynamic UI rendering

**Interface**:
```python
class LLMManager:
    def get_available_providers() -> List[str]
    def get_models_for_provider(provider: str) -> Dict[str, str]
    def get_model_parameters(model: str) -> Dict[str, Dict]
    def chat(
        provider: str,
        model: str,
        messages: List[Dict],
        system_prompt: Optional[str] = None,
        context: Optional[str] = None,       # RAG context
        temperature: float = 1.0,
        max_tokens: int = 65536,
        top_p: float = 1,
        reasoning_effort: str = 'medium',    # for GPT-5/o1/o3 only
        use_defaults: bool = False
    ) -> str
```

**Supported Models**:
- **Gemini**: `gemini-2.5-flash-lite`, `gemini-2.5-flash`, `gemini-2.5-pro`, `gemini-3-pro-preview`
- **OpenAI**: `gpt-5`, `gpt-5-nano`, `gpt-5-mini`, `gpt-5.1`, `gpt-5.1-chat-latest`, `gpt-5.2`, `gpt-5.2-pro`, `gpt-5.2-chat-latest`

**RAG Context Injection**:
```python
if context:
    context_message = f"\n\nContesto RAG:\n{context}\n\n"
    messages[0]["content"] = context_message + messages[0]["content"]
```

**Model Parameter Metadata** (`llm_manager_config.py`):
- `MODEL_PARAMETERS` dict maps model names → supported parameters + UI metadata
- Used by Compass to dynamically render LLM configuration controls
- Reasoning models (GPT-5 series): ignore temperature/max_tokens; use `reasoning_effort`

## 3.2 SessionManager (`utils/session_manager.py`)

**Purpose**: Session CRUD + BDI data persistence.

**Interface**:
```python
class SessionManager:
    def create_session(name, context, llm_provider, llm_model, description="", tags=None, llm_settings=None) -> str
    def get_session(session_id) -> Optional[Dict]
    def get_all_sessions(status=None) -> List[Dict]
    def update_session_metadata(session_id, name, description, tags, status) -> bool
    def get_bdi_data(session_id) -> Dict
    def save_bdi_data(session_id, bdi_data) -> bool
    def delete_session(session_id) -> bool
```

**Storage**: Each session is a directory at `data/sessions/{uuid}/` containing:
- `metadata.json` — name, description, tags, created_at, last_accessed, status
- `config.json` — llm_provider, llm_model, llm_settings, context
- `current_bdi.json` — live BDI data (desires + beliefs + intentions)
- `chat_history/` — per-agent chat logs (optional)

## 3.3 ContextManager (`utils/context_manager.py`)

**Purpose**: Multi-context management. Each context has its own knowledge base (ChromaDB) and belief base.

**Interface**:
```python
class ContextManager:
    def create_context(name, description="") -> Dict
    def get_all_contexts() -> List[Dict]
    def get_context(normalized_name) -> Optional[Dict]
    def update_context_metadata(normalized_name, updates) -> bool
    def delete_context(normalized_name) -> bool
    def get_belief_base_path(normalized_name) -> str
    def _normalize_name(name) -> str
```

**Storage**: Each context at `data/contexts/{normalized_name}/`:
- `context_metadata.json` — name, normalized_name, description, document_count, belief_count
- `belief_base.json` — pre-extracted beliefs (generated by Knol, consumed by Believer)
- `chroma_db/` — persistent ChromaDB vector store

## 3.4 DocumentProcessor (`utils/document_processor.py`)

**Purpose**: RAG pipeline. Handles ingestion, chunking, embedding, storage, and retrieval.

**Interface**:
```python
class DocumentProcessor:
    def __init__(context_name=None, persist_directory=None)
    def initialize_db() -> None
    def release_connections() -> None    # ⚠️ MUST be called after batch ops
    def clear_database() -> None
    def process_pdf(file) -> List[str]
    def process_text(file) -> List[str]
    def process_url(url) -> List[str]
    def add_documents(chunks, source) -> None
    def query(text, top_k=5) -> List[Dict]
```

**Pipeline Parameters**:
- **Chunking**: `RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)`
- **Embedding**: `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (384-dim, 50+ languages)
- **Storage**: ChromaDB persistent client, one collection per context

## 3.5 ConversationAuditor (`utils/auditor.py`)

**Purpose**: Quality validation meta-agent. Evaluates agent outputs and drives improvement.

**Interface**:
```python
class ConversationAuditor:
    def evaluate_response(response_text, context, module) -> Dict
    def detect_finalization(user_input, module) -> bool
    def generate_suggestions(auditor_feedback) -> List[Dict]
```

**Rubric Dimensions** (0–5 score each):
1. Query coherence
2. Module alignment (are we producing the right BDI layer?)
3. Context preservation
4. Dialogue progression
5. Beneficiary/domain focus
6. Finalization handling

**Finalization Detection**: Keyword patterns like "generate the report", "final json", "ho finito", etc.

## 3.7 PromptsManager (`utils/prompts.py`)

**Purpose**: LRU-cached prompt loader. Reads Markdown files from `prompts/` directory.

**Interface**:
```python
def get_prompt(agent_name, use_cache=True, prompt_suffix="system_prompt") -> str
def get_all_prompts(use_cache=True) -> Dict[str, str]
def clear_cache() -> None
```

**File Convention**: `prompts/{agent_name}_{prompt_suffix}.md`
**Supported Agents**: ali, believer, cuma, genius, auditor, desires_auditor, belief_auditor, intention_auditor
